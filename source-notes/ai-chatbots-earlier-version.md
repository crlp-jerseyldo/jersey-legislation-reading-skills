# AI Chatbots Earlier Version

## Status

Starter distilled note based on earlier chatbot-oriented thinking reflected in the user material.

## Main Points Carried Into The Repo

- Earlier chatbot framing is useful mainly as a record of user-facing risks.
- The biggest risk is not only hallucination, but also wrong source state, wrong date, weak citation, and false certainty.
- A modular skills repository is safer than a single-answer product because it forces a reading workflow rather than hiding it.
- Future evaluation should compare chatbot-style outputs against skill-driven outputs, especially on source control and stopping behaviour.

## Repo Impact

This note mainly supports the decision to structure the repository around reusable reading operations instead of conversational answer generation.
